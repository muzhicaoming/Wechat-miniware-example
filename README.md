# 微信小程序示例项目 (WeChat MiniProgram Example)

## 项目简介

这是一个纯前端的微信小程序示例项目，展示了9个常用功能模块。项目采用纯前端架构，所有数据存储在本地，不依赖后端服务。

## 项目架构

### 技术栈
- **框架**: 微信小程序原生框架
- **存储**: 微信小程序本地存储 (wx.setStorageSync)
- **架构模式**: 纯前端架构，无后端依赖

## 快速开始

### 环境要求
- 微信开发者工具
- Node.js (用于运行测试)

### 安装步骤

1. **克隆项目**
   ```bash
   git clone <repository-url>
   cd wechat-miniware-example
   ```

2. **安装依赖** (可选，仅用于测试)
   ```bash
   npm install
   ```

3. **打开项目**
   - 使用微信开发者工具打开项目目录
   - 使用测试号或正式AppID进行开发

4. **运行测试** (可选)
   ```bash
   npm test
   npm run test:coverage
   ```

## 开发说明

### 本地存储
所有数据使用微信小程序的本地存储API (`wx.setStorageSync`) 进行存储，包括：
- 用户登录信息
- 待办事项列表
- 备忘录数据
- 应用设置

### 登录状态管理
登录状态通过 `utils/auth.js` 进行管理，所有状态存储在本地，不会与服务器交互。

### 数据持久化
- 数据存储在微信小程序本地存储中
- 清除应用数据会删除所有本地存储的信息
- 不支持云端同步

## 测试

## 部署

### 微信小程序发布

详细的发布指南请参考：[微信小程序发布指南](./docs/WECHAT_DEPLOYMENT.md)

#### 快速步骤
1. **注册小程序账号**
   - 访问 [微信公众平台](https://mp.weixin.qq.com/) 注册小程序
   - 获取AppID

2. **配置AppID**
   - 在 `project.config.json` 中配置AppID
   - 或在微信开发者工具中设置

3. **上传代码**
   - 在微信开发者工具中点击"上传"
   - 填写版本号和项目备注

4. **提交审核**
   - 在微信公众平台"版本管理"中提交审核
   - 上传小程序页面截图
   - 填写版本描述

5. **发布上线**
   - 审核通过后点击"发布"
   - 小程序正式上线

### 注意事项
- 确保所有功能正常运行
- 检查是否有敏感信息泄露
- 确保符合微信小程序规范
- 测试各种场景下的功能
- 选择正确的服务类目
