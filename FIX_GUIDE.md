# 修复指南：恢复九宫格页面

## 问题说明

项目配置被改成了云开发模板配置，指向了 `miniprogram/` 目录，导致显示的是云开发快速启动页面，而不是我们创建的九宫格页面。

## 解决方案

### 方法1：在微信开发者工具中重新打开项目（推荐）

1. **关闭当前项目**
   - 在微信开发者工具中关闭当前项目

2. **重新打开项目**
   - 点击"项目" -> "新建项目" 或 "导入项目"
   - 选择项目目录：`/Users/qitmac001395/workspace/QAL/wechat-miniware-example`
   - **重要**：确保选择的是项目根目录，不是 `miniprogram/` 子目录
   - AppID: `wxa510a56a488e9c1b`
   - 点击"导入"

3. **验证配置**
   - 打开 `project.config.json`
   - 确认没有 `miniprogramRoot` 字段
   - 确认 `appid` 是 `wxa510a56a488e9c1b`

4. **编译运行**
   - 点击"编译"按钮
   - 应该能看到九宫格首页

### 方法2：修改项目配置

如果项目已经打开：

1. **检查配置**
   - 打开 `project.config.json`
   - 确认配置已恢复（我已经帮你恢复了）

2. **重新编译**
   - 在微信开发者工具中点击"编译"按钮
   - 或者按 `Cmd+R` (Mac) / `Ctrl+R` (Windows)

3. **清除缓存**
   - 如果还是显示云开发页面，尝试：
     - 点击"工具" -> "清除缓存" -> "清除全部"
     - 然后重新编译

### 方法3：删除云开发目录（可选）

如果上述方法都不行，可以删除云开发相关目录：

```bash
# 备份后删除（如果需要保留可以重命名）
cd /Users/qitmac001395/workspace/QAL/wechat-miniware-example
mv miniprogram miniprogram.backup
mv cloudfunctions cloudfunctions.backup
```

然后重新编译项目。

## 如果还是不行

如果按照上述步骤操作后还是显示云开发页面：

1. **检查文件路径**
   - 确认微信开发者工具打开的是项目根目录
   - 不是 `miniprogram/` 子目录

2. **检查 app.json**
   - 打开根目录的 `app.json`
   - 确认第一行是 `"pages/index/index"`（不是云开发的页面）

3. **重新创建项目**
   - 如果以上都不行，可以：
     - 备份当前代码
     - 在微信开发者工具中创建新项目
     - 将我们的代码复制过去
